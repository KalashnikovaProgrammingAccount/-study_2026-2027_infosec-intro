# Цель работы

Получить навыки создания разделов на диске и файловых систем. Получить навыки
монтирования файловых систем.

# Задание

1. Добавить два диска на виртуальной машине
2. Продемонстрировать навыки создания разделов MBR с помощью fdisk 
3. Продемонстрировать навыки создания логических разделов с помощью fdisk 
4. Продемонстрировать навыки создания раздела подкачки с помощью fdisk 
5. Продемонстрировать навыки создания разделов GPT с помощью gdisk 
6. Продемонстрировать навыки форматирования файловой системы XFS 
7. Продемонстрировать навыки форматирования файловой системы EXT4 
8. Продемонстрировать навыки ручного монтирования файловых систем 
9. Продемонстрировать навыки монтирования файловых систем с помощью /etc/fstab
10. Выполнить задание для самостоятельной работы 


# Выполнение лабораторной работы

Создаем два диска на виртуальной машине размером 512 МБ (рис. [-@fig:001]).

![Диски](image/1.png){#fig:001 width=70%}

Посмотрим перечень разделов на всех имеющихся в системе устройствах жёстких дисков и сделаем разметку диска /dev/sdb с помощью ути-
литы fdisk (рис. [-@fig:002]).

![Разметка](image/2.png){#fig:002 width=70%}

Далее получим справку по командам m, проверим сколько дискового пространства p и добавим новый раздел n, далее создадим новый p. После посмотрим тип раздела и нажмем w , чтобы записать изменения на диск и выйти из fdisk (рис. [-@fig:003]).

![Диск](image/3.png){#fig:003 width=70%}

Таблица разделов находится у нас только в памяти ядра. Сравним вывод следующих команд fdisk -l /dev/sdb и cat /proc/partitions (рис. [-@fig:004]).

![Сравнение](image/4.png){#fig:004 width=70%}

Запишем изменения при помощи команды partprobe /dev/sdb (рис. [-@fig:005]).

![Сравнение](image/5.png){#fig:005 width=70%}

В терминале с полномочиями администратора запустите fdisk /dev/sdb и введем n , чтобы добавить новый раздел. Далее введием e , чтобы создать расширенный раздел и выполним ряд других команд, после чего запишем изменения (рис. [-@fig:006]).

![Изменение](image/6.png){#fig:006 width=70%}

Запустим fdisk, нажмем n , чтобы добавить новый раздел, далее нажмем Enter, чтобы принять первый сектор по умолчанию, затем изменим тип раздела. Для этого нажмите t. После создания логического раздела введите w , чтобы записать изменения на диск. Чтобы завершить процедуру и обновить таблицу разделов ядра, введем partprobe /dev/sdb (рис. [-@fig:007]).

![Изменение](image/7.png){#fig:007 width=70%}

Просмотрим информацию о добавленных разделах: cat /proc/partitions и 
fdisk --list /dev/sdb (рис. [-@fig:008]).

![Просмотр](image/8.png){#fig:008 width=70%}

fdisk -l показывает таблицу разделов на диске (MBR/GPT), включая расширенные и логические разделы.

cat /proc/partitions показывает только разделы, загруженные в ядро как блочные устройства.

Разница: fdisk видит все разделы, включая расширенные (sdb2), а /proc/partitions не показывает расширенные разделы, только основные и логические (sdb1, sdb5).

Далее отформатируем раздел подкачки, используя команду mkswap /dev/sdb6 и для включения вновь выделенного пространства подкачки используем swapon /dev/sdb6. Для просмотра размера пространства подкачки, которое в настоящее время выделено, введем free -m (рис. [-@fig:009]).

![Просмотр](image/9.png){#fig:009 width=70%}

![Сохранение](image/10.png){#fig:010 width=70%}

![Сохранение](image/11.png){#fig:011 width=70%}

Посмотрим разделы на втором добавленном вами ранее диске /dev/sdc:
gdisk -l /dev/sdc. Создадим раздел с помощью gdisk: gdisk /dev/sdc. Введем n , чтобы добавить новый раздел, далее нажмием Enter, чтобы принять предлагаемый по умолчанию первый сектор, затем установим тип раздела. Нажмем p , чтобы отобразить разбиение диска(рис. [-@fig:012]).

![Сохранение](image/12.png){#fig:012 width=70%}

Обновим таблицу разделов: partprobe /dev/sdc и просмотрим информацию о добавленных разделах: cat /proc/partitions и gdisk -l /dev/sdc (рис. [-@fig:011]).

![Обновление](image/13.png){#fig:013 width=70%}

В терминале с полномочиями администратора для диска dev/sdb1 создадим файловую систему XFS и установим метки файловой системы в xfsdisk используйте команду (рис. [-@fig:014]).

![Создание](image/14.png){#fig:014 width=70%}

В терминале с полномочиями администратора для диска dev/sdb5 создайте файловую систему EXT4. Далее для установки метки файловой системы в ext4disk используем команду tune2fs -L ext4disk /dev/sdb5. Для установки параметров монтирования по умолчанию для файловой системы используйте команду tune2fs -o acl,user_xattr /dev/sdb5 (рис. [-@fig:015]).

![Установка](image/15.png){#fig:015 width=70%}

Для ручной установки файловой системы используется команда mount. Чтобы отключить смонтированную файловую систему, используется команда umount. Для создания точки монтирования для раздела введем mkdir -p /mnt/tmp. Далее смонтируем файловую систему и отмонтируем раздел, а также проверим, что раздел отмонтирован: mount (рис. [-@fig:016]).

![Проверка](image/16.png){#fig:016 width=70%}

Создадим точку монтирования для раздела XFS /dev/sdb1и посмотрим информацию об идентификаторах блочных устройств (UUID). Далее введием blkid /dev/sdb1 и скопируем значение идентификатора UUID для устройства /dev/sdb1. Откройте файл /etc/fstab на редактирование и добавьте следующую строку: UUID=значение_идентификатора /mnt/data xfs defaults 1 2 (рис. [-@fig:017]).

![Создание](image/17.png){#fig:017 width=70%}

Проверим конфигурацию и следующая команда монтирует всё, что указано
в /etc/fstab: mount -a. Проверим, что раздел примонтирован правильно: df -h (рис. [-@fig:018]).

![Проверка](image/18.png){#fig:018 width=70%}

Добавим две партиции на диск с разбиением GPT. Создадим оба раздела размером 100 MiB. Один из этих разделов настроим как пространство подкачки, другой раздел отформатируем файловой системой ext4. Настроим сервер для автоматического монтирования этих разделов. Установим раздел ext4 на /mnt/data-ext и установите пространство подкачки в качестве области подкачки (рис. [-@fig:019]).

# Контрольные вопросы  

1. Какой инструмент используется для создания разделов GUID?

Ответ: gdisk для разделов GUID

2. Какой инструмент применяется для создания разделов MBR?

Ответ: fdisk - для раздлелов MBR

3. Какой файл используется для автоматического монтирования разделов во время загрузки?

Ответ: /etc/fstab - для автоподключения при загрузке 

4. Какой вариант монтирования целесообразно выбрать, если необходимо, чтобы файловая система не была автоматически примонтирована во время загрузки?

Ответ: использовать опцию монтирования  noaouto

5. Какая команда позволяет форматировать раздел с типом 82 с соответствующей файловой системой?

Ответ: тип 82 это Linux swap

6. Вы только что добавили несколько разделов для автоматического монтирования при загрузке. Как можно безопасно проверить, будет ли это работать без реальной перезагрузки?

Ответ: проверить без перезагрузки sudo mount -a

7. Какая файловая система создаётся, если вы используете команду mkfs без какой либо спецификации файловой системы?

Ответ: по умолчанию ext2

8. Как форматировать раздел EXT4?

Ответ: mkfs.ext4 /deb/...

9. Как найти UUID для всех устройств на компьютере?

Ответ: командой  blkid
